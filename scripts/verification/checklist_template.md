# 技能验证Checklist模板
# 版本：v1.0
# 适用：所有技能
# 责任人：技能开发者

## 📋 验证Checklist

### 1️⃣ 基础验证（必选）
- [ ] **文件存在验证**
  - SKILL.md文件存在且可读
  - 验证命令：`ls -la $SKILL_PATH/SKILL.md`
  - 通过标准：文件存在且非空

- [ ] **语法正确性**
  - Markdown/YAML语法无错误
  - 验证命令：检查frontmatter格式
  - 通过标准：`---`分隔符正确

- [ ] **元数据完整**
  - 包含name和description字段
  - 验证命令：`grep -c '^name:'` 和 `grep -c '^description:'`
  - 通过标准：两个字段都存在

### 2️⃣ 功能验证（按类型选择）

#### 开发类技能
- [ ] **构建脚本验证**
  - 构建脚本存在且可执行
  - 验证命令：`[ -f build.sh ] && bash build.sh`
  - 通过标准：构建成功或无构建脚本

- [ ] **测试脚本验证**
  - 测试脚本存在且通过
  - 验证命令：`[ -f test.sh ] && bash test.sh`
  - 通过标准：测试通过或无测试脚本

- [ ] **文档完整性**
  - README文档存在且>10行
  - 验证命令：`wc -l README.md`
  - 通过标准：行数≥10

#### 文档类技能
- [ ] **模板文件验证**
  - 模板目录包含至少1个模板
  - 验证命令：`find templates -name '*.md' | wc -l`
  - 通过标准：数量≥1

- [ ] **生成功能验证**
  - 文档生成脚本能正常运行
  - 验证命令：`timeout 30 bash generate.sh sample`
  - 通过标准：成功运行或无生成脚本

- [ ] **Git集成验证**
  - 包含Git上传相关功能
  - 验证命令：`grep -c 'git.*push\|git.*commit'`
  - 通过标准：至少1处引用

#### 质量类技能
- [ ] **检查清单验证**
  - 包含检查清单文件
  - 验证命令：`[ -f checklist.md ] && wc -l checklist.md`
  - 通过标准：行数≥5

- [ ] **风险分析验证**
  - 包含风险分析内容
  - 验证命令：`grep -ci '风险' SKILL.md`
  - 通过标准：至少1处

- [ ] **验证点覆盖**
  - 包含明确的验证点
  - 验证命令：`grep -ci '验证\|verify\|check' SKILL.md`
  - 通过标准：至少3处

### 3️⃣ 集成验证
- [ ] **依赖检查**
  - 所有依赖项明确列出
  - 验证命令：检查依赖声明
  - 通过标准：依赖清晰

- [ ] **环境兼容性**
  - 技能支持目标环境
  - 验证命令：检查平台声明
  - 通过标准：环境要求明确

- [ ] **错误处理**
  - 包含错误处理机制
  - 验证命令：检查错误处理代码
  - 通过标准：有错误处理逻辑

### 4️⃣ 性能验证
- [ ] **响应时间**
  - 关键操作响应时间合理
  - 验证命令：测量执行时间
  - 通过标准：<30秒

- [ ] **资源使用**
  - 内存和CPU使用合理
  - 验证命令：监控资源消耗
  - 通过标准：无内存泄漏

- [ ] **并发能力**
  - 支持合理的并发访问
  - 验证命令：压力测试
  - 通过标准：能处理并发请求

### 5️⃣ 安全验证
- [ ] **输入验证**
  - 所有输入都经过验证
  - 验证命令：检查输入处理
  - 通过标准：有输入验证

- [ ] **权限控制**
  - 权限控制合理
  - 验证命令：检查权限检查
  - 通过标准：有权限控制

- [ ] **数据保护**
  - 敏感数据得到保护
  - 验证命令：检查数据处理
  - 通过标准：数据保护措施

## 🔄 验证流程

### 1. 预验证准备
- 确保技能目录结构正确
- 备份当前状态
- 准备测试数据

### 2. 执行验证
- 按顺序执行验证点
- 记录每个验证结果
- 收集验证输出

### 3. 结果分析
- 计算总分和等级
- 识别问题根本原因
- 制定改进计划

### 4. 问题修复
- 按优先级修复问题
- 重新验证修复结果
- 更新验证记录

### 5. 报告生成
- 生成验证报告
- 提交给相关方
- 归档验证结果
